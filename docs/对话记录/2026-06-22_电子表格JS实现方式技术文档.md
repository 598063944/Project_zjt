## 二、Luckysheet 引擎（当前实现）

### 2.1 引入背景

Luckysheet 是一个开源的纯前端电子表格库，API 风格参考了 Google Sheets 和 Excel Online。选择它作为首版实现的原因：

- **本地运行**：所有 JS 文件打包在 `luckysheet/dist/` 目录，无需联网
- **API 简洁**：`luckysheet.create()` / `luckysheet.getLuckysheetfile()` 即可完成初始化和数据读取
- **中文支持**：原生 `lang: 'zh'` 配置

### 2.2 文件结构

```
luckysheet/
├── luckysheet.html              ← 主入口 HTML（170 行）
├── qwebchannel.js               ← QWebChannel JS 库（16 KB）
└── dist/
    ├── luckysheet.umd.js        ← 核心引擎（3 MB，UMD 格式）
    ├── css/
    │   └── luckysheet.css       ← 样式
    ├── assets/
    │   └── iconfont/            ← 图标字体
    └── plugins/
        └── js/plugin.js         ← 插件（公式、图表等）
```

### 2.3 HTML 初始化流程（[luckysheet.html](luckysheet/luckysheet.html)）

```javascript
// 1. 加载 QWebChannel，获取 bridge 对象
new QWebChannel(qt.webChannelTransport, function(ch) {
    bridge = ch.objects.bridge;
    window.bridge = bridge;
    // 2. 初始化 Luckysheet（空表格或挂起数据）
    createLuckysheet(null);
});

// 3. 创建表格实例
function createLuckysheet(data) {
    luckysheet.create({
        container: 'luckysheet',
        data: data || [defaultSheet],
        allowEdit: true,
        showtoolbar: true,
        lang: 'zh',
        hook: {
            cellUpdated: function(r, c, oldValue, newValue, isRefresh) {
                bridge.handle_js_action('cell_changed', JSON.stringify({row: r, col: c}));
            }
        }
    });
}
```

### 2.4 暴露的全局 JS 函数

| 函数 | 方向 | 作用 |
|------|------|------|
| `window.loadWorkbookData(sheetData)` | Python → JS | 加载完整工作簿（Luckysheet 格式的数组） |
| `window.setCellValue(row, col, value, sheetIndex)` | Python → JS | 设置单个单元格值 |
| `window.getAllCellData()` | Python → JS | 读回所有数据，返回 JSON 字符串 |
| `window.setEditable(editable)` | Python → JS | 启用/禁用编辑 |

### 2.5 Luckysheet 数据格式

```json
[{
  "name": "Sheet1",
  "celldata": [
    {"r": 0, "c": 0, "v": {"v": "姓名", "m": "姓名"}},
    {"r": 0, "c": 1, "v": {"v": "年龄", "m": "年龄"}},
    {"r": 1, "c": 0, "v": {"v": "张三", "m": "张三"}},
    {"r": 1, "c": 1, "v": {"v": 25, "m": "25"}}
  ],
  "row": 100,
  "column": 20,
  "status": 1
}]
```

特点：
- **顶层是数组**，每个元素代表一个 sheet
- `celldata` 是 **稀疏扁平数组**（非二维矩阵），只存有值的格子
- `status: 1` 标记当前活动 sheet
- 公式通过 `v.f` 字段存储（如 `"=SUM(B2:B10)"`）

## 三、Univer 引擎（候选实现）

### 3.1 引入背景

Univer（0.21.1）是新一代开源电子表格引擎，架构更现代（ES Module + TypeScript）。在 `dist/` 打包目录中保留了独立的 Univer 实现，用于对比评估。

### 3.2 文件结构

```
dist/数据批量处理工具/_internal/univer/
└── univer.html                  ← Univer 入口 HTML（全量内联）
```

**注意**：Univer 的 JS 资源全部从 CDN 加载（`esm.sh`），未打包到本地。

### 3.3 HTML 初始化流程（[univer.html](dist/数据批量处理工具/_internal/univer/univer.html)）

```javascript
import { UniverSheetsCorePreset } from 'https://esm.sh/@univerjs/preset-sheets-core@0.21.1';
import { createUniver } from 'https://esm.sh/@univerjs/presets@0.21.1';
import { UniverInstanceType } from 'https://esm.sh/@univerjs/core@0.21.1';

function initUniver() {
    var preset = UniverSheetsCorePreset({ container: document.getElementById('app') });
    var result = createUniver({ presets: [preset] });
    univerInstance = result.univer;
    univerAPI = result.univerAPI;
    univerAPI.createUnit(UniverInstanceType.UNIVER_SHEET, {});
}
```

### 3.4 Univer 数据格式

```json
{
  "sheets": {
    "sheet1": {
      "name": "Sheet1",
      "cellData": {
        "0": { "0": {"v": "姓名"}, "1": {"v": "年龄"} },
        "1": { "0": {"v": "张三"}, "1": {"v": 25} }
      }
    }
  },
  "activeSheet": "sheet1"
}
```

特点：
- **顶层是对象**（key 为 sheet ID）
- `cellData` 是**二维嵌套对象**（行号 → 列号 → 单元格）
- 键均为字符串（"0", "1", ...），需 `int()` 转换
- 公式通过 `cellData[row][col].f` 存储

## 四、两种格式的桥接层

### 4.1 为什么需要转换

Python 端的**内部统一格式**采用 Univer 的 `cellData` 二维对象结构（便于 openpyxl 读写、DataFrame 转换）。但当前前端引擎是 Luckysheet，其数据格式是扁平数组。因此需要双向转换。

### 4.2 转换函数（[spreadsheet_page.py:366-431](spreadsheet_page.py#L366-L431)）

#### `_univer_to_luckysheet(workbook_data) -> list`

**输入**：Univer 格式 `{"sheets": {"id": {"name": ..., "cellData": {"0": {"0": {"v": ...}}}}}}`  
**输出**：Luckysheet 格式 `[{"name": ..., "celldata": [{"r": ..., "c": ..., "v": {...}}]}]`

核心逻辑：
```python
for sheet_id, info in workbook_data.get("sheets", {}).items():
    celldata = []
    for rk, row in info.get("cellData", {}).items():
        ri = int(rk)
        for ck, cell in row.items():
            ci = int(ck)
            v_obj = {"v": cell.get("v"), "m": str(cell.get("v"))}
            if "f" in cell:
                v_obj["f"] = cell["f"]
            celldata.append({"r": ri, "c": ci, "v": v_obj})
    sheets.append({"name": info.get("name"), "celldata": celldata, ...})
```

#### `_luckysheet_to_univer(sheets) -> dict`

**输入**：`getAllCellData()` 返回的 Luckysheet 数组  
**输出**：Univer 格式的 `workbookData`

核心逻辑：
```python
for sheet in sheets:
    cell_data = {}
    for cell in sheet.get("celldata", []):
        rk = str(cell.get("r", 0))
        ck = str(cell.get("c", 0))
        cell_info = {}
        if v.get("v") is not None: cell_info["v"] = v["v"]
        if v.get("f"): cell_info["f"] = v["f"]
        cell_data.setdefault(rk, {})[ck] = cell_info
```

### 4.3 转换在数据流中的位置

```
Python 内部格式 (Univer cellData)
       │
       ▼ _univer_to_luckysheet()
Luckysheet 格式 → JSON → runJavaScript → JS loadWorkbookData()
       │
       ▼ 用户编辑
JS getAllCellData() → Luckysheet 格式 → handle_js_action()
       │
       ▼ _luckysheet_to_univer()
Python 内部格式 (Univer cellData) → openpyxl 保存 / DataFrame 处理
```

**设计意义**：Python 端所有存储和处理逻辑只认 Univer 格式，Luckysheet 相关逻辑被**完全封装**在转换函数和 `SpreadsheetBridge` 中。如果未来切换到 Univer 引擎，只需删除转换层，其余代码无需改动。

## 五、SpreadsheetBridge 通信机制

### 5.1 架构（[spreadsheet_page.py:570-661](spreadsheet_page.py#L570-L661)）

```
┌──────────────────────────────────────────────────┐
│                  Python 端                        │
│                                                    │
│  SpreadsheetBridge(QObject)                        │
│  ├── actionReceived = pyqtSignal(str, str)  # 出站 │
│  ├── load_workbook_data(data)                      │
│  │   └── _run_js("window.loadWorkbookData(...)")   │
│  ├── request_cell_data()                           │
│  │   └── _run_js("window.getAllCellData()...")     │
│  └── handle_js_action(action, payload)  # 入站     │
│      └── actionReceived.emit(action, payload)      │
│                                                    │
│  QWebChannel ─── registerObject("bridge", bridge)  │
└──────────────────────────────────────────────────┘
                        ↕
┌──────────────────────────────────────────────────┐
│                  JS 端 (Luckysheet)                │
│                                                    │
│  window.bridge = channel.objects.bridge;           │
│                                                    │
│  window.loadWorkbookData(data)  ← Python 调用     │
│  window.getAllCellData()        ← Python 调用     │
│  bridge.handle_js_action(...)   → Python 回调     │
└──────────────────────────────────────────────────┘
```

### 5.2 Python → JS 方向

通过 `page.runJavaScript(js_code)` 直接注入，**不经过 QWebChannel**。

```python
def _run_js(self, js: str):
    if self._page:
        self._page.runJavaScript(js)
```

### 5.3 JS → Python 方向

通过 QWebChannel 的 `@pyqtSlot` 回调：

```python
@pyqtSlot(str, str)
def handle_js_action(self, action: str, payload_json: str):
    """统一回调入口"""
    self.actionReceived.emit(action, payload_json)
```

### 5.4 Action 类型一览

| Action | 方向 | 触发时机 | Payload |
|--------|------|----------|---------|
| `page_ready` | JS → Py | 表格引擎初始化完成 | `{}` |
| `cell_changed` | JS → Py | 用户编辑了单元格 | `{"row": r, "col": c}` |
| `cell_data_response` | JS → Py | `getAllCellData()` 返回结果 | Luckysheet JSON 数组 |
| `data_loaded` | JS → Py | `loadWorkbookData()` 完成 | `{"success": true, "sheets": n}` |
| `formula_error` | JS → Py | 公式执行错误 | 错误信息 |
| `selection_changed` | JS → Py | 选区变化（预留） | — |

## 六、本地 HTTP 服务器

### 6.1 为什么需要（[spreadsheet_page.py:42-81](spreadsheet_page.py#L42-L81)）

QWebEngineView 的 `setHtml()` 方法将 HTML 注入为 `data:` URL 或临时文件。在此模式下，`<script type="module">`（ES Module）**无法加载外部 CDN 资源**，且本地 `file://` 协议也有限制。

解决方案：在 `127.0.0.1:18900-18999` 上启动一个 daemon 线程的轻量 HTTP 服务器，将 `luckysheet/` 目录作为静态文件根目录，通过 `http://127.0.0.1:{port}/luckysheet.html` 加载。

```python
def _start_local_http(directory: str) -> int:
    """在 directory 上启动本地 HTTP 服务器，返回端口号"""
    for port in range(18900, 19000):
        try:
            httpd = _TCPServer(("127.0.0.1", port), handler)
            _server_port = port
            threading.Thread(target=httpd.serve_forever, daemon=True).start()
            return port
        except OSError:
            continue
```

### 6.2 端口复用

- `_server_port` 全局变量，首次启动后缓存端口号，后续调用直接返回
- `allow_reuse_address = True` 避免端口 TIME_WAIT 问题
- 端口范围 `18900-18999`，避开常用端口

## 七、后台线程架构

### 7.1 Worker 线程一览

| Worker | 文件 | 作用 |
|--------|------|------|
| `_ImportWorker` | spreadsheet_page.py:93 | 后台读取 XLSX → Univer cellData |
| `_MysqlQueryWorker` | spreadsheet_page.py:179 | 执行 MySQL SQL → DataFrame → cellData |
| `_MysqlListTablesWorker` | spreadsheet_page.py:236 | 获取 MySQL 表列表（用于导入对话框） |

### 7.2 _ImportWorker 详细流程

```
1. openpyxl.load_workbook(path, data_only=False, read_only=True)  ← 提取公式
2. openpyxl.load_workbook(path, data_only=True, read_only=True)   ← 提取缓存值
3. 遍历每个 sheet：
   3.1 构建 cached_values 字典（data_only=True 的值）
   3.2 遍历每个 cell：
       - 如果是公式（str, 以 '=' 开头）→ {"f": formula, "v": cached_value}
       - 否则 → {"v": value}
   3.3 存入 cell_data[row_key][col_key]
4. 发射 finished 信号 → 包装为 Univer 格式
```

**关键设计**：
- `read_only=True` 流式读取，避免大文件 MemoryError
- 双工作簿策略（公式 + 缓存值），确保公式单元格既有公式字符串也有上次计算值
- 在子线程运行，不阻塞 UI

### 7.3 _dataframe_to_cell_data_static（[spreadsheet_page.py:283-362](spreadsheet_page.py#L283-L362)）

将 pandas DataFrame 转为 Univer cellData 格式的静态函数（线程安全）。

类型转换链：
```
numpy.int64     → int()
numpy.float64   → float() (NaN/Inf → None)
numpy.bool_     → bool()
datetime/date   → str()
Decimal         → float()
bytes           → utf-8 decode / hex()
ndarray         → tolist()
其他            → str()（兜底）
```

第一行（row 0）固定为表头（`df.columns`），数据从 row 1 开始。

### 8.3 延迟加载机制

编辑器页面采用**懒加载**：只有用户双击文件进入编辑器时才创建 QWebEngineView（`_ensure_spreadsheet_editor()`）。HTML 加载是异步的，因此设计了两级挂起队列：

```python
# 挂起数据（页面未就绪时）
self._spreadsheet_pending_data = None

# 挂起文件（编辑器未初始化时）
self._spreadsheet_pending_file = None
```

在 `page_ready` 事件中依次消费：

```python
def _on_spreadsheet_js_action(self, action, payload_json):
    if action == 'page_ready':
        self._spreadsheet_ready = True
        if self._spreadsheet_pending_data is not None:
            self._spreadsheet_bridge.load_workbook_data(self._spreadsheet_pending_data)
            self._spreadsheet_pending_data = None
        if hasattr(self, '_spreadsheet_pending_file') and self._spreadsheet_pending_file:
            pf = self._spreadsheet_pending_file
            self._spreadsheet_pending_file = None
            self._import_file_to_editor(pf)
```

## 九、Luckysheet vs Univer 对比评估

| 维度 | Luckysheet | Univer |
|------|-----------|--------|
| **加载方式** | 本地 UMD 文件（3 MB） | CDN ES Module（需联网） |
| **加载速度** | 快（本地文件） | 慢（CDN 网络依赖） |
| **离线可用** | ✅ 完全离线 | ❌ 需要网络 |
| **版本** | 无活跃维护（已停更） | 0.21.x 持续迭代 |
| **功能** | 基础表格 + 简单公式 | 完整公式引擎 + 协作支持 |
| **API 设计** | `luckysheet.create()` / `getLuckysheetfile()` | `createUniver()` / `univerAPI` |
| **数据格式** | 扁平数组（稀疏） | 嵌套对象（二维） |
| **setEditable** | pointerEvents + opacity 控制 | 未实现（空函数） |
| **社区** | 44K+ Star，已停更 | 活跃开发中 |

## 十、文件 I/O 完整流程

### 10.1 导入 XLSX

```
用户点击 [📥 导入 XLSX]
  → QFileDialog 选文件
  → _ImportWorker.start()（子线程）
     → openpyxl 读取（read_only=True）
     → 构建 Univer cellData 格式
     → finished 信号
  → _on_import_xlsx_done()
     → spreadsheet_load_data(workbook_data)
        → bridge.load_workbook_data()
           → _univer_to_luckysheet()  ← 转为 Luckysheet 格式
           → JSON 序列化
           → runJavaScript("window.loadWorkbookData(...)")
```

### 10.2 导出 XLSX

```
用户点击 [📤 导出 XLSX]
  → bridge.request_cell_data()
     → runJavaScript("window.getAllCellData()...")
  → JS 返回 Luckysheet JSON
  → handle_js_action('cell_data_response', ...)
     → _luckysheet_to_univer(raw)  ← 转回 Univer 格式
     → 缓存到 _spreadsheet_last_data
  → QFileDialog 选保存路径
  → openpyxl 写入 XLSX
     → 遍历 sheets → cellData → ws.cell(row, col, value)
     → 公式单元格写入 f 字段（"=SUM(...)"）
  → wb.save() + wb.close()
```

### 10.3 MySQL 导入

```
用户点击 [🗄️ MySQL导入]
  → MysqlImportDialog（选择表/自定义 SQL）
  → _MysqlQueryWorker.start()（子线程）
     → pymysql 执行 SQL
     → pd.DataFrame(rows)
     → _dataframe_to_cell_data_static(df) → Univer cellData
     → finished 信号
  → _on_mysql_import_done()
     → spreadsheet_load_data(workbook_data)
     → 同导入 XLSX 后半段
```

### 10.4 自动保存

```
用户点击 [← 返回列表]
  → switch_to_spreadsheet_list()
     → _auto_save_current_file()
        → bridge.request_cell_data()
        → QTimer.singleShot(1000, _do_auto_save)
           → openpyxl 写入 _spreadsheet_current_file
```

| 验证 openpyxl 读写兼容性 | — | 测试 |

**结论**：内部数据格式已经是 Univer 格式，转换层是纯适配代码。切换引擎的工作量约 **2-3 天**，核心风险在于 Univer 的 CDN 加载是否可在离线环境下工作。

## 十二、与其他 JS 表格实现的关系

本项目中有 **三处** 使用 JS 表格引擎，均为 `QWebEngineView + QWebChannel` 架构：

| 模块 | 引擎 | 文件 | 用途 |
|------|------|------|------|
| 电子表格 | Luckysheet | `spreadsheet_page.py` + `luckysheet/` | 通用表格编辑器 |
| 多维表格 | AG Grid | `bitable.py` + `bitable/` | MySQL 报表的交互式查看（分页/排序/筛选） |
| BI 图表 | ECharts | `dashboard.py` + `echarts/` | 数据可视化图表 |

三者共享相同的通信模式（Bridge + QWebChannel），但数据格式和业务逻辑完全独立。
