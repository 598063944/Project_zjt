try:

### 2026-07-07: 设置→CRM设置 contains/not_contains 查找字段无法通过 API 过滤

**问题**: DeviceObj（设备对象）的"产品"字段（device_product_id）是查找字段（lookup），CRM 存储的是产品记录 ID，界面上显示的是产品名称（如 "13.012.0063-00..."）。`LIKE "13.012"` 在记录 ID 上无法匹配。

**根因**: `_build_api_filters_from_settings` 中将 contains/not_contains 作为 API filter 发给 CRM，但 LIKE 操作符在查找字段上作用于记录 ID 而非显示名，导致返回 0 条。而客户端侧过滤（`_obj_query_filter_rows`）已正确通过 `__r.name` 获取产品名称做子串匹配，只是 API 层提前过滤掉了数据。

**修改文件**: `object_query.py`（1处）

`_build_api_filters_from_settings` 中 contains/not_contains 分支：不再构建 API filter，改为存入 `client_filters`，由客户端侧过滤处理：

```python
client_filters.append({
    'field': field_name,
    'field_api': api_field,
    'operator': op,
    'value': value_text,
})
```

**验证**: `python -m py_compile object_query.py` 通过

**经验**: 查找字段的显示名不能直接用于 API 过滤（LIKE 作用于记录 ID）。此类过滤应用客户端侧处理，通过 `__r.name` 获取参照对象的显示名做子串匹配。
