 # 踩坑记录 & 复利方法 — Project_001
 
 ## 用途说明
 记录本项目中踩过的坑、犯过的错误，以及经过验证的可复用方法。
 每次修改前 / 新对话前，Codex 必须先阅读本文档。
 
 分类标签说明：
 - [错误] — 明显的错误决策或实现
 - [踩坑] — 因环境、工具、依赖等外部因素导致的意外问题
 - [弯路] — 走了不必要的绕路，效率低下但方向没错
 - [复用方法] — 验证有效、值得反复使用的经验/模板/模式
