### 2026-06-21: [可复用方法] 弹窗无边框化 — 全局统一方案

**问题**：原生 `QDialog` / `QMessageBox` / `QInputDialog` 会显示系统标题栏，与项目浅色无边框风格冲突。

**解决方案**：
1. `CenteredPopupDialog` 基类添加 `FramelessWindowHint`（覆盖所有继承子类）
2. 原生 `QDialog` 实例创建后立即加 `setWindowFlags(... | FramelessWindowHint)`
3. `QMessageBox` / `QInputDialog` 禁止直接使用，改用 `common.py` 辅助函数：

```python
from common import frameless_input_text, frameless_input_getitem, frameless_message_box

# 替代 QInputDialog.getItem
item, ok = frameless_input_getitem(parent, '标题', '提示', items_list)

# 替代 QMessageBox.warning/information/critical
frameless_message_box(parent, '标题', '内容')

# 替代 QMessageBox.question
result = frameless_message_box(parent, '确认', '内容',
    QMessageBox.StandardButton.Yes | QMessageBox.StandardButton.No,
    QMessageBox.StandardButton.No)
```

## 2026-06-21 Qt 样式表优先级挤占 + exec() 模态阻断外部关闭

### 背景

自定义报表 → 编辑报表 → 排序，弹出 SortConfigDialog（custom_report.py），两个 bug：
1. 每行排序条件的「✕ 删除」按钮完全不可见
2. 点击弹窗外部不会关闭对话框

### 踩坑经过

**Bug 1 — 删除按钮不可见：**
1. 删除按钮 del_btn 设了 `setFixedSize(28, 28)` + 内联样式 `QPushButton { padding: ... }`
2. 但全局样式表（Project_zjt_3.5.8.py L1438）设了 `QDialog QPushButton { padding: 6px 16px; }`
3. Qt 样式表优先级：后代选择器 `QDialog QPushButton` > 普通 `QPushButton`
4. 全局的 `padding: 6px 16px`（水平 32px）超过按钮 28px 宽度，文字被完全挤出

**Bug 2 — 点外部不关闭：**
1. CenteredPopupDialog 的 close_on_outside 依赖两个机制：(a) QApplication 级事件过滤器捕获 MouseButtonPress (b) WindowDeactivate 事件
2. SortConfigDialog 用 `dlg.exec()` 打开 → 模态
3. Qt 模态循环在 `notify()` 之前阻断父窗口的鼠标事件
4. 父窗口的 MouseButtonPress 到不了事件过滤器
5. Windows 上点击被阻断的父窗口也不会触发 WindowDeactivate
6. 两个机制全部失效

### 根因

**Bug 1 — 选择器优先级：**
```python
# ❌ 内联样式优先级低于全局后代选择器
del_btn.setStyleSheet("QPushButton { padding: 4px; }")
# 全局: QDialog QPushButton { padding: 6px 16px; }  ← 胜出

# ✅ 用 objectName 提升优先级
del_btn.setObjectName("sortDelBtn")
del_btn.setStyleSheet("QPushButton#sortDelBtn { padding: 0px; }")
# QPushButton#id 优先级 > QDialog QPushButton  ← 胜出
```

**Bug 2 — exec() 模态阻断：**
```python
# ❌ exec() 模态 → 父窗口事件被阻断 → 事件过滤器收不到
dlg = SortConfigDialog(columns, config, self)
dlg.exec()  # ApplicationModal，父窗口完全阻断

# ✅ 重写 exec()：非模态 show + 本地事件循环
def exec(self):
    self.show()                        # 非模态 → 父窗口正常接收事件
    parent.installEventFilter(self)    # 父窗口事件过滤器生效
    QTimer.singleShot(150, self._arm_outside)
    loop = QEventLoop(self)
    self.accepted.connect(loop.quit)
    self.rejected.connect(loop.quit)
    loop.exec()                        # 阻塞等待，但父窗口事件正常分发
    return self.result()
```

### 复利要点

> **Qt 样式表优先级速查（从低到高）：**
> 1. `QPushButton { ... }` — 最低
> 2. `QDialog QPushButton { ... }` — 后代选择器
> 3. `QPushButton#myBtn { ... }` — ID 选择器（objectName）
> 4. `!important` — 最高但不推荐
>
> **小尺寸按钮（< 60px）必须用 `setObjectName()` + `QPushButton#id` 写样式，否则会被全局 `QDialog QPushButton` 的 padding 挤没。**

> **`QDialog.exec()` 是模态的 — Qt 模态循环在 `notify()` 之前阻断父窗口鼠标事件，导致 QApplication 级和父窗口级事件过滤器都收不到外部点击。**
>
> 需要外部关闭时的正确模式：
> 1. 重写 `exec()` 为非模态 `show()` + 本地 `QEventLoop`
> 2. 在父窗口上 `installEventFilter` 捕获鼠标事件
> 3. 延迟 150ms 激活外部关闭标志（防止刚打开就关闭）
> 4. `closeEvent` 中清理事件过滤器和事件循环

### 2026-06-28: [错误] 可用数据表刷新按钮未清除编辑器字段缓存

**问题**：自定义报表 → 编辑报表 → 可用数据表，点击"刷新"按钮后，表格列表的字段数据和行数未更新，仍显示旧值。

**根因**：
- TableSelectorPanel.refresh() 每次都会清空 _table_data 并从 MySQL 重新查询（包括 SHOW TABLES、SHOW COLUMNS、SELECT COUNT(*)），这部分逻辑正确
- 但 ReportEditorPage._get_cleaned_table_fields() 有一个 _cleaned_fields_cache 本地缓存字典
- 刷新按钮只更新了 TableSelectorPanel._table_data，没有通知编辑器清除其本地缓存
- 编辑器下次获取字段时（如点击画布卡片展开字段列表），直接命中 _cleaned_fields_cache 中的旧数据，导致显示的字段列表和行数是旧的

**解决方案**：
- TableSelectorPanel.refresh() 末尾添加 self._emit_refreshed() 调用
- 新增 _emit_refreshed() 方法：沿父控件链向上查找 _on_table_selector_refreshed 并调用
- ReportEditorPage 新增 _on_table_selector_refreshed()：清除 _cleaned_fields_cache + 标记 _fields_fresh = False

**验证**：py_compile 通过

**经验教训**：
1. 两层缓存必须有统一的失效机制：TableSelectorPanel._table_data 和 ReportEditorPage._cleaned_fields_cache
2. 跨组件缓存一致性要求：修改一个组件的缓存时，必须通知依赖该缓存的组件刷新
3. 查找父控件链 parent() 比使用 Qt 信号更轻量，适合简单的一次性通知场景
